> 状态：**当前有效基线**
> 最后更新：2026-07-11
> 目的：把“现在真正要做成什么”从旧规划、创意稿、已实现代码中抽离出来，作为后续迭代与验收的统一依据。

---

## 0. 这份文档解决什么问题

当前仓库同时存在三类信息源：

1. **早期蓝图 / 远期规划**
2. **序章创意设计稿（偏理想目标）**
3. **当前 Godot 项目中已经实现并能跑通测试的版本**

三者已经出现明显漂移。

因此，从现在开始：

> **序章 1~4 的继续开发，以本文件 + 当前主线工程实现为基线。**

旧文档仍保留参考价值，但在与本文件冲突时，**本文件优先**。

## 3. 当前序章纵切片的总目标

序章 1~4 当前不是要证明“全项目做完了”，而是要证明：

> **这个项目已经具备一个可玩的、可测试的、具有统一气质的 ASOIAF × FE 式序章纵切片。**

当前纵切片必须满足四个核心要求：

1. **四章可完整串通**
2. **基础战棋机制可教学、可验证、可重复**
3. **地图风格、UI、对话、演出至少达到同一审美方向**
4. **后续做第一幕时，可以把这一套流程直接复用**

---

## 4. 当前锁定的风格基线

### 4.1 战场视觉

当前已锁定为：

- **庄重、暗色、非 Q 版**
- 全章节统一使用**程序化暗色地形渲染**
- 主战斗场景已移除旧 `TileMapLayer` 与旧瓦片贴图依赖
- 地图可读性主要靠：
  - 地形底色差异
  - 墙体/水体/桥梁细节
  - 高亮层
  - UI 与危险区提示

### 4.2 角色表现

当前锁定为：

- 对话系统支持角色立绘
- 立绘资源当前以 **96×96 风格化头像** 为基线
- 当前不是最终正式美术，但已脱离“抽象像素块”阶段

### 4.3 UI 方向

当前锁定为：

- 深色战场 UI
- 中文字体统一可显示
- 战斗信息优先可读性，而非复杂装饰

## 7.1 序章一（当前基线）

### 当前实现角色定位

- **机械角色：** 纯教学关 / 第一接触关
- **玩法重点：**
  - 选中单位
  - 移动范围
  - 攻击范围
  - 战斗预测
  - 结束回合
  - 胜利目标格概念

### 当前实现内容

- 地图：10×8
- 我方：奈德、霍兰德
- 敌方：3 名皇家卫兵（弱化数值）
- 胜利条件：
  - 奈德到达北侧目标格 `Vector2i(5, 0)`
  - 或敌军全灭
- 战后：进入 `prologue_1_post.json`，推进到第 2 章

### 当前基线中的问题与说明

这是**当前最不稳定的一章**。

现存冲突：

- `Opening.gd` 的三段开场过场在讲**篡夺者战争起义**
- 但 `prologue_1_pre.json` / `prologue_1_post.json` 的对话却是在讲**奈德赴任君临、入城受阻**
- 这意味着序章一当前存在**时间线与剧情语境不一致**的问题

### 因此，当前基线对 Ch1 的锁定方式是：

先锁定它的**机械职能**，暂不把叙事稿视为最终定稿：

> **Ch1 当前的真实基线是“10×8 的战斗教学关”，其叙事包装仍处于待统一状态。**

后续如要重写，应优先保持：

- 小地图教学规模
- 奈德主导
- 简单敌军构成
- 明确的目标格胜利体验

而不是一上来改复杂机制。

## 7.2 序章二《三叉戟》（当前基线）

### 当前实现角色定位

- **机械角色：** 第一张真正的战术地图
- **叙事角色：** 篡夺者战争决战章节
- **玩法主题：** 三桥推进 / 南北岸分层 / 击破关键敌将

### 当前实现内容

- 开场标题卡：序章·二《三叉戟》
- 开场过场：`ch2_opening.json`
- 地图：28×20
- 我方：劳勃 + 3 名反叛领主
- 敌方：雷加、巴里斯坦、塔格利安士兵
- 地图结构：
  - 北岸王军防线
  - 中部三桥河道
  - 南岸泥泞集结区

### 当前胜利链（已实现并测试）

- 雷加死亡
  - 触发 `ch2_rhaegar_fall.json`
  - 进入战后流程 `_on_won_ch2()`
  - 播放 `ch2_split.json`
  - 播放 `ch2_post.json`
  - 推进到第 3 章

### 当前锁定结论

> **Ch2 是当前序章中“战斗结构最清楚”的一章，可作为后续战术地图模板参考。**

后续优化重点应放在：

- 三桥差异进一步拉开
- 雷加决战的演出记忆点强化
- 南北岸视觉与战术层次继续加强

## 7.3 序章三《极乐塔》（当前基线）

### 当前实现角色定位

- **机械角色：** 强调“不是所有地图都靠歼灭解决”
- **叙事角色：** 极乐塔事件、莱安娜伏笔
- **玩法主题：** 向北推进 / 突破守门 Boss / 到达目标格触发事件

### 当前实现内容

- 开场标题卡：序章·三《极乐塔》
- 开场过场：`ch3_opening.json`
- 地图：24×18
- 我方：奈德、霍兰、北境骑士若干
- 敌方：亚瑟·戴恩 + 多恩骑士
- 亚瑟·戴恩当前被设置为：
  - 关键守门 Boss
  - 原地守卫（`move = 0`）

### 当前胜利链（已实现并测试）

- 奈德到达目标格 `Vector2i(12, 2)`
- 触发 `_trigger_ch3_tower()`
  - 播放 `ch3_dayne_trigger.json`
  - 移除亚瑟·戴恩
  - 播放 `ch3_lyanna.json`
  - 播放 `ch3_post.json`
  - 推进到第 4 章

### 当前锁定结论

> **Ch3 的核心不是“击败亚瑟”，而是“顶着压力推进到塔门，触发不可战胜之人的剧情性崩解”。**

后续优化重点应放在：

- 亚瑟·戴恩的压迫感呈现
- 通往目标点的节奏设计
- 过场与战斗衔接的情绪重量

---

## 7.4 序章四《铁王座》（当前基线）

### 当前实现角色定位

- **机械角色：** 序章规模最大的一章 / 攻城与多层防线模板
- **叙事角色：** 君临陷落、兰军观望、詹姆事件、劳勃登基
- **玩法主题：** 从南向北攻入红堡，穿透河流、城墙、内院与王座区

### 当前实现内容

- 开场标题卡：序章·四《铁王座》
- 开场过场：`ch4_opening.json`
- 先进入部署界面：`DeployScreen_Ch4.tscn`
- 地图：36×26
- 当前地图结构已锁定包含：
  - 红堡内院
  - 内护城河
  - 外墙 / 内城墙 / 南城墙
  - 黑水河 / 外护城河
  - 三组桥位
  - 中央大道与南部部署区

### 当前势力结构

- 我方：奈德 + 最多 4 名北境骑士（部署选择）
- 中立：兰尼斯特军（team=2，观望态度）
- 敌方：残余王军 + 王军指挥官

### 当前关键事件链

1. 开场说明兰军中立
2. 向北推进
3. 击杀王军指挥官
4. 播放 `ch4_lannister_join.json`
5. 兰军从地图移除（表现为政治尘埃落定，而非加入可控战斗）
6. 自动进入结局链
7. 若尚未触发过詹姆事件，则补播詹姆相关对话 / 过场
8. 播放 `ch4_post.json`
9. 播放 `ch4_ending.json`
10. 返回 Opening（章节重置到 1）

### 当前对 Ch4 胜利条件的真实定义

名义上仍保留“奈德到达铁王座格”的触发入口，
但当前主设计实际上已经转为：

> **击杀王军指挥官 = 解决战场主要悬念 = 自动流向结局。**

这是当前实现层面的有效设计，不再要求玩家在高潮后手动走最后几格去“补触发”。

### 当前锁定结论

> **Ch4 是当前序章纵切片的气质标杆地图。**

后续优化重点应放在：

- 攻城节奏是否清楚
- 城墙 / 河流 / 桥梁 / 内院可读性
- 王军 → 兰军 → 王座结局的叙事推进是否足够顺

## 8. 当前锁定的内容表达方式

### 8.1 对话

- 继续使用 JSON 驱动
- 继续以短对话 + 旁白内心段落为主
- 当前文风方向已经偏向：
  - 克制
  - 短句
  - 少解释
  - 重情绪留白

### 8.2 过场

- 继续使用 JSON slide 过场
- 当前已支持简单 `scene_art` 类型切换
- 当前定位是“轻量叙事过场”，不是完整动画系统

### 8.3 地图事件

- 继续优先使用 `BattleBootstrap.gd` 中的章节分发与事件触发
- 不建议把序章主线逻辑分裂回多个旧章节脚本中
