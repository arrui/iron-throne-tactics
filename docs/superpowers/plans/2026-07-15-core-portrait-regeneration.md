# Core Portrait Regeneration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the approved eight core character portrait assets with new TV-recognizable, tactical-RPG-style portraits while preserving existing runtime filenames, portrait mappings, and a fully passing Godot validation run.

**Architecture:** Keep runtime integration unchanged: replace only the existing `game/冰与火/assets/units/*_portrait.png` files and leave `BattleBootstrap.gd` / dialogue mappings untouched. Store one selected high-resolution master portrait per character plus a prompt/export log in a project-owned design workspace so future portrait batches can reuse the same style language without redoing discovery work. Execute in two slices: first lock the style with Ned Stark + Rhaegar Targaryen, then expand the remaining six portraits using the same crop, lighting, and material rules.

**Tech Stack:** OpenAI image generation workflow via the `imagegen` skill; PNG portrait assets; Python 3 + Pillow for local review-sheet/contact-sheet generation; Godot 4 headless validation via `./scripts/test.sh`.

## Global Constraints

- Do not rename or remap any portrait filenames in `game/冰与火/assets/units/`; runtime code must continue to use the current `UNIT_PORTRAIT_MAP` in `game/冰与火/scripts/battle/BattleBootstrap.gd`.
- Final runtime portraits must remain square PNGs and satisfy the existing automated requirement of at least `96x96`; for this batch export exactly `96x96`.
- Keep one selected high-resolution master PNG per character in-repo; do not leave project-owned source art only in `$CODEX_HOME/generated_images/*` or any temp directory.
- Style target is the approved hybrid finish: TV-version-recognizable identity + mature tactical RPG painted lighting/materials; avoid both abstract/icon-like simplification and pure publicity-still photorealism.
- First establish the style with `ned_stark_portrait.png` and `rhaegar_targaryen_portrait.png`, then expand the remaining six approved portraits.
- All asset changes must pass `./scripts/test.sh` and must not regress portrait loading in battle, dialogue, or deploy flows.
- Do not modify map tiles, map sprites, dialogue UI structure, or portrait-system logic in this phase unless validation proves a code change is strictly required.
- Commit only the selected master portrait per character plus the final runtime portrait; discard unselected drafts before the final feature commit.
- If `godot --headless --import` refreshes matching `.import` metadata for changed PNGs, include those `.import` diffs in the same task commit as the PNGs.

## Global style stem {#global_style}

Use this stem for every generation call before adding the character-specific section.

```text
Use case: stylized-concept
Asset type: tactical RPG in-game character portrait master
Primary request: create a high-resolution bust or half-body portrait for a grim low-fantasy tactics game, designed to crop cleanly to a readable 96x96 in-game portrait
Subject: one adult character from the HBO-style Game of Thrones adaptation, recognizable from the TV version while still feeling like premium game portrait art rather than a photo still

Lighting/mood: dramatic but controlled key light, restrained contrast, strong facial planes, readable materials, war-torn atmosphere, not glossy poster lighting
Background: very simple soft-focus background with low detail only, color-field or subtle environmental haze that supports the character and never competes with the face
Constraints: one subject only; no text; no watermark; no logo; no frame; no exaggerated fantasy magic; no abstract icon look; preserve age, hair, beard, and social rank cues; the face must remain the first read at 96x96
Avoid: anime, chibi, flat-shaded mobile icon, caricature, celebrity photo shoot, modern fashion, extra people, busy battle tableau, tiny face, over-rendered jewelry, oversaturated colors, soft blurry eyes, smeared facial features
```

- [ ] **Step 4: Build an enlarged anchor contact sheet for 96x96 readability review**

```bash
python3 - <<'PY'
from PIL import Image, ImageOps, ImageDraw
from pathlib import Path
base = Path('game/冰与火/assets/units')
out = Path('design/portraits/core-batch-2026-07-15/review/anchors_contact_sheet.png')
items = [
    ('Ned', 'ned_stark_portrait.png'),
    ('Rhaegar', 'rhaegar_targaryen_portrait.png'),
]
thumb_scale = 4
label_h = 28
pad = 16
font_fill = (230, 230, 230)
bg = (20, 22, 28)
card = (36, 39, 48)
thumb = 96 * thumb_scale
sheet_w = pad + len(items) * (thumb + pad)
sheet_h = thumb + label_h + pad * 2
canvas = Image.new('RGB', (sheet_w, sheet_h), bg)
draw = ImageDraw.Draw(canvas)
for i, (label, filename) in enumerate(items):
    x = pad + i * (thumb + pad)
    y = pad
    tile = Image.open(base / filename).convert('RGB').resize((thumb, thumb), Image.Resampling.NEAREST)
    canvas.paste(ImageOps.expand(tile, border=2, fill=card), (x, y))
    draw.text((x, y + thumb + 6), label, fill=font_fill)
out.parent.mkdir(parents=True, exist_ok=True)
canvas.save(out)
print(out)
PY
```
