# 序章最小兵种系统设计（Phase 0 方案）

> 日期：2026-07-17  
> 适用范围：`game/冰与火` 当前序章 1~4 纵切片  
> 目标：在**不推翻现有可玩基线**的前提下，把项目从“能打通”升级为“阵容有差异、角色有身份、演出可扩展”的高质量序章 Demo。

---

## 1. 为什么现在必须先做兵种系统

当前项目已经具备：

- 可跑通的四章主流程
- 战斗预测、地形、耐久、道具、AI、存档等战棋底层
- 章节部署、对话、过场、标题卡、测试体系

但目前仍存在 4 个直接影响品质的问题：

1. **角色在战斗中的身份感不强**：很多单位只是“不同名字 + 一组数值”。
2. **出战选择缺乏价值**：尤其是第四章，北境骑士模板重复，选择更像形式而非战术。
3. **战斗动画无法系统扩展**：如果没有职业分层，后续很难建立“共享职业动画 + 主角/Boss 专属强化”的资产管线。
4. **立绘与地图打磨缺乏落点**：角色美术和地图视觉细化需要先知道“这个单位/这个区域的战术身份是什么”。

因此，当前阶段不该先做“完整大职业树”，而应该先做：

> **序章最小兵种系统（Prologue Minimal Class System, PMCS）**

它必须先解决三个问题：

- 玩家一眼能看懂“这人是干什么的”
- 出战编组时真的有取舍
- 后续动画 / 立绘 / AI / UI 都有统一扩展锚点

---

## 2. 这套系统不做什么

为了控制范围，本阶段**明确不做**：

- 不做完整转职树
- 不做成长率 / 升级分支重构
- 不做弓、法术、飞兵、龙、骑射等完整中后期系统
- 不做复杂技能树 / 被动树
- 不做 Fire Emblem 全量职业相克矩阵
- 不做全关卡重构

本阶段只做：

1. **职业身份层**
2. **武器三角的体验强化**
3. **部署阵容差异化（优先 Ch4）**
4. **为动画 / 立绘 / UI 预留标准字段**

---

## 3. 设计原则

### 3.1 先可读，再复杂

序章 Demo 最重要的不是“职业很多”，而是：

- 玩家第一次看就能判断谁是前锋、谁是守线、谁是突击、谁是决斗单位
- 玩家在第四章部署界面能做真正选择

### 3.2 先做“职业模板”，不做“角色技能爆炸”

优先让单位通过这些维度形成差异：

- 武器类型
- 数值倾向
- 移动力
- 护甲轻重
- 战术职责
- 动画家族

而不是一上来给每个角色塞专属技能。

### 3.3 与现有代码兼容，渐进接入

当前项目已经有：

- `weapon_type`
- `weapon_rank`
- `class` 文本字段
- 剑 / 枪 / 斧三角

所以新的最小系统必须：

- 可在现有 `UnitData` 结构上增量接入
- 不要求一次性重写所有旧 JSON
- 允许通过默认映射逐步迁移

### 3.4 兵种系统必须为演出服务

这套系统不只是数值分类，还要为这些后续工作服务：

- 共享战斗动画家族
- 立绘轮廓与装备语言
- 地图 sprite silhouette
- UI 图标和部署简介
- 章节地图战术结构

---

## 4. 当前基线约束（基于现有工程）

截至当前版本，项目已经具备以下事实：

### 4.1 已存在的武器三角

代码中已经实现：

- 剑 > 斧 > 枪 > 剑
- 当前数值修正：`ATK ±1` / `HIT ±5`

这意味着本阶段无需“发明新克制系统”，而是要把它从：

> “底层公式存在”

强化为：

> “玩家能感知到，并会据此编组和进攻”

### 4.2 当前单位数据分布

现有单位已天然分成几类：

- 奈德：领袖型剑士
- 霍兰：灵活枪兵
- 劳勃 / 义军将领 / 兰军：斧系重击单位
- 王军 / 塔格利安士兵：枪系线列步兵
- 雷加 / 多恩骑士：枪骑与冲击单位
- 巴里斯坦 / 王军指挥官 / 詹姆（未来）：剑系精英/王卫
- 亚瑟·戴恩：传奇剑圣 Boss

也就是说，兵种雏形**其实已经存在于数据里**，只是没有被系统化地定义、展示和利用。

### 4.3 当前最大痛点在 Ch4 部署

目前第四章部署界面本质上是：

- 奈德固定
- 其余候选多为同模板北境骑士

所以玩家面对的不是“阵容选择”，而是“数量选择”。

这必须优先修正。

---

## 5. 序章最小兵种系统：总结构

本方案将兵种系统拆成 4 层：

1. **武器家族**：剑 / 斧 / 枪
2. **职业模板**：决定数值倾向、职责、动画家族
3. **移动 / 护甲标签**：决定表现与后续扩展方向
4. **角色实例**：具体单位 JSON 与章节配置

其中，真正需要稳定下来的核心，是第 2 层：**职业模板**。

---

## 6. 武器三角：本阶段策略

### 6.1 保留现有底层公式

本阶段建议**保留现有数值公式**作为第一版实施基线：

- 优势：`ATK +1 / HIT +5`
- 劣势：`ATK -1 / HIT -5`

原因：

- 当前测试已覆盖该逻辑
- 现有系统可用，改动小
- 当前最急需的是“让玩家感知到”，不是先调大倍率

### 6.2 强化玩家感知，而不是先改数值

本阶段应补上以下表现：

- 战斗预测中明确显示：`克制 / 被克制 / 均势`
- 部署界面卡片增加武器图标或文案标签
- 战斗动画中加入轻量武器差异反馈
  - 剑：快、利、切入
  - 枪：直线突刺、守线
  - 斧：慢、重、破阵
- 新手提示或 Ch4 编组提示明确说明三角关系

### 6.3 下一阶段再决定是否加大数值权重

如果实装后玩家仍感知较弱，再考虑在 Phase 1 的平衡阶段讨论：

- `HIT ±10`
- 增加更明显的颜色/文案反馈
- 为特定职业加入“克制时额外收益”

但这不应是 Phase 0 的优先事项。

它们刚好覆盖了序章当前真实需求：

- 主角定位：`lord_blade`
- 轻快辅助：`scout_spear`
- 杂兵骨架：`line_spear` / `raider_axe`
- 冲击单位：`lancer_elite`
- 精英剑士：`duelist_sword`
- 守门 Boss：`heavy_guard_sword` / `legend_swordmaster`
- 中层敌将 / 友将：`captain_axe`

这已经足够支撑：

- Ch1 教学
- Ch2 决战三桥
- Ch3 塔前门神
- Ch4 攻城编组

## 8. 每个职业模板的设计定义

以下定义不是精确数值，而是**平衡方向**与**资产制作约束**。

### 8.1 `lord_blade` 北境领主

**代表：** 奈德  
**作用：** 玩家主控核心，既不能太脆，也不能像 Boss 那样碾压。

**数值倾向：**

- HP：高
- Pow：高
- Spd：中上
- Def：高
- Move：中

**战术定位：**

- 中轴推进
- 对斧系稳定压制
- 适合作为“必须活着抵达”的目标单位

**演出关键词：**

- 克制、沉稳、指挥感
- 不花哨，但每一下都重

---

### 8.2 `scout_spear` 游猎枪手

**代表：** 霍兰  
**作用：** 用来拉开与普通枪兵的差异，体现“轻、快、准”。

**数值倾向：**

- HP：中低
- Pow：中
- Spd：高
- Def：低
- Move：中上

**战术定位：**

- 侧翼穿插
- 收残血
- 协助打开通道

**演出关键词：**

- 快刺、轻步、灵敏

---

### 8.3 `line_spear` 线列枪兵

**代表：** 王军、塔格利安士兵  
**作用：** 序章最基础的“阵线单位”。

**数值倾向：**

- HP：中
- Pow：中
- Spd：低中
- Def：中
- Move：低中

**战术定位：**

- 守桥、守门、站线
- 对剑系有天然威慑

**演出关键词：**

- 正面、规整、纪律感

---

### 8.4 `lancer_elite` 精英枪骑

**代表：** 雷加、多恩骑士  
**作用：** 让枪兵里出现“真正会冲击你的单位”。

**数值倾向：**

- HP：中高
- Pow：高
- Spd：中上
- Def：中
- Move：高

**战术定位：**

- 冲击侧翼
- 强迫玩家重视布阵
- 作为中层 Boss / 精英单位的主要模板

**演出关键词：**

- 冲锋、突刺、穿透力

---

### 8.5 `raider_axe` 重击斧兵

**代表：** 劳勃、兰尼斯特士兵、部分北境候选  
**作用：** 最直观的高伤害职业模板。

**数值倾向：**

- HP：高
- Pow：高
- Spd：低中
- Def：中高
- Move：中

**战术定位：**

- 撕开枪兵线
- 强攻破阵
- 适合作为“打穿中轴”的主力

**演出关键词：**

- 抡、砸、沉重、震屏

---

### 8.6 `captain_axe` 战团统领

**代表：** 义军将领  
**作用：** 介于普通斧兵和主角之间的副核模板。

**数值倾向：**

- HP：中高
- Pow：中高
- Spd：中
- Def：中
- Move：中

**战术定位：**

- 副线推进
- 带头压上
- 比普通斧兵更稳定

**演出关键词：**

- 重击，但比劳勃更稳、更军官化

---

### 8.7 `duelist_sword` 王卫决斗者

**代表：** 巴里斯坦、詹姆  
**作用：** 让剑系精英与奈德区分开来，强调“快、准、贵族精英”。

**数值倾向：**

- HP：中
- Pow：中高
- Spd：高
- Def：中
- Move：中或中高

**战术定位：**

- 斩杀关键单位
- 对斧系压制明显
- 用于塑造精英骑士压迫感

**演出关键词：**

- 快速连斩、精确、利落

---

### 8.8 `heavy_guard_sword` 重卫统领

**代表：** 王军指挥官  
**作用：** 作为关底门槛单位，强调“难推开”。

**数值倾向：**

- HP：高
- Pow：高
- Spd：低中
- Def：高
- Move：低

**战术定位：**

- 堵门
- 站点
- 作为 Ch4 最终守关者

**演出关键词：**

- 沉、硬、压迫、守门

---

### 8.9 `legend_swordmaster` 传奇剑圣

**代表：** 亚瑟·戴恩  
**作用：** 不是普通职业单位，而是“事件型 Boss 模板”。

**数值倾向：**

- HP：中高
- Pow：高
- Spd：极高
- Def：中高
- Move：可为 0（原地门神）

**战术定位：**

- 剧情门神
- 极高的个体压迫
- 必须通过事件 / 机制 / 强叙事来解决

**演出关键词：**

- 传奇、纯净、锋利、压倒性技术感

## 9. 移动与护甲标签（最小扩展）

为了让职业模板能接到后续动画和地图设计上，本阶段建议增量加入两个标签层。

### 9.1 `move_type`

候选值：

- `foot`：步行
- `cavalry`：骑乘
- `guard`：重装守卫（本阶段可仍按步行处理，只作为标签）

### 9.2 `armor_type`

候选值：

- `light`
- `medium`
- `heavy`

### 9.3 这两个标签本阶段的作用

本阶段不一定立刻接入移动公式差异，但必须先有标签，用于：

- 立绘和 sprite 轮廓控制
- 战斗动画家族分配
- UI 文案和图标展示
- 后续地图地形针对性设计

## 10. 数据结构建议（兼容现有工程）

### 10.1 新增字段建议

建议在单位 JSON 中逐步引入：

```json
{
  "class_id": "lord_blade",
  "move_type": "foot",
  "armor_type": "medium",
  "animation_family": "lord_sword"
}
```

### 10.2 与现有字段的关系

- 现有 `class`：保留，继续作为玩家可见的职业名/文本名
- 新增 `class_id`：系统逻辑主键
- 现有 `weapon_type` / `weapon_rank`：保留
- 新增 `move_type` / `armor_type` / `animation_family`：为后续资产与体验服务

### 10.3 渐进接入策略

为避免一次性改全量数据：

1. 先创建 `ClassCatalog.gd`，统一维护职业模板
2. 如果 JSON 中没有 `class_id`，则按旧 `class` 或文件名推断默认值
3. 新增 / 重做的单位优先写显式 `class_id`

这样可以先改 Ch4 候选 roster，不会一次炸全工程。

## 11. 第四章部署重构：从“4 个同模板骑士”改为“5 选 4 的真实编组”

这是本阶段最重要的落地变化。

### 11.1 设计目标

第四章的部署必须做到：

- 每个候选单位一眼能看出职责
- 玩家必须在“中轴突破 / 守桥 / 侧翼 / 收割 / 抗压”之间取舍
- 阵容搭配能明显影响推进体验

### 11.2 改造原则

当前不是要做 20 个职业，而是把当前 5 个“北境骑士占位槽”改成：

> **5 个职业明确、职责明确、立绘未来可区分的北境候选**

玩家从中选择 4 个出战，1 个留守。

### 11.3 建议的 5 名候选（第一版）

> 说明：这里先给出**战术身份稿**。姓名、立绘、台词可在 Phase 1 美术与文本细化时再锁定。

| roster_id | 暂定显示名 | class_id | 武器 | 主要职责 | 建议数值倾向 |
|---|---|---|---|---|---|
| north_axebreaker | 破门斧卫 | raider_axe | 斧 | 破枪阵 / 中轴打穿 / 拆守线 | 高 HP / 高 Pow / 低中 Spd |
| north_spearwall | 白港枪卫 | line_spear | 枪 | 守桥 / 抗冲 / 护侧 | 高 Def / 中 HP / 低中 Move |
| north_swiftsword | 林地快剑 | duelist_sword | 剑 | 清杂 / 追击 / 打斧系 | 高 Spd / 中 Pow / 低 Def |
| north_rider | 侧翼游骑 | lancer_elite 或 captain_axe（视资产决定） | 枪或斧 | 抢点 / 补位 / 追杀后排 | 中高 Move / 中高 Spd |
| north_veteran | 持旗老兵 | lord_blade 或 captain_axe 的弱化版 | 剑 | 稳定副核 / 扛线 / 填补阵容缺口 | 均衡，容错高 |

### 11.4 为什么是这 5 类

这样一来，Ch4 的部署选择就会出现真实取舍：

- 想稳守黑水桥：必带 `north_spearwall`
- 想快速穿中轴：优先 `north_axebreaker`
- 想解决侧翼兰军/斧兵干扰：更需要 `north_swiftsword`
- 想增强机动：考虑 `north_rider`
- 想提高整体容错：可选 `north_veteran`

这会把部署从：

> “带几个人”

变成：

> “我准备用什么方法攻下君临”

### 11.5 Ch3 也应提前埋一点预告

为了避免 Ch4 突然出现“你怎么一下子有这么多不同兵种”，建议在 Ch3 先让北境随行者出现**轻微差异化预告**：

- 至少把 3 个北境骑士从同模板改成 2~3 种倾向
- 让玩家在塔前战看到“北境并非一色模板兵”

## 12. 各章节的兵种落位建议

### Ch1 风暴地教学关

- 奈德：`lord_blade`
- 霍兰：`scout_spear`
- 王军：`line_spear`

**教学重点：**

- 让玩家理解剑 / 枪关系
- 奈德是“稳推进主核”，霍兰是“轻快辅助”

### Ch2 三叉戟

- 劳勃：`raider_axe`
- 义军将领：`captain_axe`
- 雷加：`lancer_elite`
- 巴里斯坦：`duelist_sword`
- 塔格利安士兵：`line_spear`

**战术观感：**

- 义军斧系破北岸枪线

## 13. 这套兵种系统如何服务战斗动画

后续战斗动画不应按“每个角色都单独做整套”，而应按下面结构推进：

### 13.3 这样做的好处

- 主角/Boss 又能明显更“值钱”

## 14. 这套兵种系统如何服务立绘与地图美术

### 14.1 立绘

每个职业模板都要有清晰轮廓关键词：

- `lord_blade`：狼皮 / 厚披肩 / 冷色 / 沉稳
- `scout_spear`：轻甲 / 窄肩 / 快 / 湿地感
- `line_spear`：规整、制式、阵线感
- `raider_axe`：厚重、前压、力量感
- `duelist_sword`：贵族、精细、锋利
- `heavy_guard_sword`：重甲、守门、压迫
- `legend_swordmaster`：传奇、纯净、几乎神话感

### 14.2 地图

地图打磨时，关键区域应服务职业身份：

- 枪兵区：桥、门、窄口
- 斧兵区：破墙、城门、街巷冲突点
- 剑士区：侧翼廊道、内院、决斗空间
- Boss 区：地标、轴线尽头、镜头焦点

这会让地图细化不再只是“多加点像素”，而是更有战术语义。
