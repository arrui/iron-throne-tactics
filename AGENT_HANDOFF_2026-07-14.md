## 目前仍然缺什么

### 高优先级

- 统一战斗风格还只完成了顶部 HUD 和 Ch4 部署界面，战斗内其他浮层仍未完全纳入同一主题：
  - `ActionMenu`
  - `PredictPanel`
  - `ResultPanel`
  - `SupportPopup`
  - `GameOver`
- 章节过场和战斗外 UI 的风格统一还不彻底：
  - `ChapterTransition`
  - 对话框 `DialogueBox`
  - 可能还包括章节 Opening 过场容器
- 地图图块语言虽然已经完成一轮重设计，但还没有进入“精修版”阶段：
  - 同一地形的边缘处理
  - 城墙/建筑细节复用规则
  - 桥梁、河道、门洞的视觉语义一致性
  - 更细的地表装饰层

### 中优先级

- 缺少真正的 CI 接入；当前测试主要依赖本地命令行运行。
- 缺少面向 UI 截图或像素级 diff 的视觉回归机制。
- 缺少“从 Opening 跑完整章链路”的更高层端到端回归包装。
- 缺少更系统的资源清单文档，方便继续扩展美术资产时不破坏统一风格。

## 建议下一个 agent 从哪里开始

建议不要再回头做大而散的重规划，直接沿着已经形成的视觉基线继续推进。

### 第一优先级：补完战斗内子面板主题统一

目标：把战斗界面从“顶部 HUD 已统一”推进到“所有战斗浮层同一语言”。

建议顺序：

1. `ActionMenu`
2. `PredictPanel`
3. `ResultPanel`
4. 为这三块补结构和样式断言测试

这样做的价值：

- 直接提升战斗时的整体一致性
- 风险低，改动集中
- 最容易继续扩展现有自动化测试资产

### 第二优先级：把共享主题扩展到战斗外 UI

建议顺序：

1. `ChapterTransition`
2. `DialogueBox`
3. `SupportPopup`
4. `GameOver`

目标是建立“章节标题卡 -> 对话 -> 部署 -> 战斗 HUD -> 结果弹层”完整一致的视觉链路。

### 第三优先级：地图图块进入精修阶段

当前地图已经从玩法语义和关卡可玩性上站住脚，下一步不应该推翻重做，而是做“统一图块语言”的精修。

建议方式：

- 先整理每种地形的视觉规则表，再动图块绘制逻辑。
- 优先统一 Ch1-Ch4 共用语义：平原、森林、墙体、门洞、河流、桥梁、建筑。
- 每次只动一类语义，并补对应回归测试，避免再次出现章节间风格割裂。

## 建议执行流程

每轮迭代都按这个顺序执行：

1. 只做一小块明确 UI 或地图目标
2. 补测试或更新现有断言
3. 运行全量测试
4. 再继续下一块
